[ Title ]              ____________________

The quest gets appended to the selected zone’s quest array
Metadata updates in your Journal view
Optionally saved to localStorage or exported to file

# 🔧 CODEx Update - Ruff Notes
# Focus: Full CRUD Implementation + Contextual Routing Fixes + Zone Integration

## Issues to Resolve:

1. Universal CRUD Capabilities
- [ ] Ensure all panels (Zones, Lore, Achievements, Journal) support: Add, Edit, Soft Delete, Restore.
- [ ] Deleted content must remain restorable (undo/archive).
- [ ] Future: Support full app re-entry of all data via Add/Edit UI for QA validation.

2. Panel-Specific Edit Routing
- [ ] Fix: Clicking "Edit" routes generically to Add/Edit page.
- [ ] Update: Edit logic must detect content type (Zone, Lore, Achievement, etc.).
- [ ] Save logic must send content to correct panel — not default everything to Quest Journal.

3. Zone CRUD Platform
- [ ] Create standalone Zone Editor Modal/Panel:
      - Add/Edit/Delete zone entries
      - Manage zone quests by category (Main, Side, Repeatable)
      - Track task statuses (Upcoming, In Progress, Completed)
- [ ] Ensure UI updates and data persistence for zone changes.

4. Zone → Quest Journal Sync
- [ ] Selecting a Zone in “All Zones” view should feed its quests into Quest Journal.
- [ ] Journal entries should be grouped under the zone name for clarity.

5. Quest Status Logic
- [ ] Fix: Deactivate currently deletes quests.
- [ ] Update: Deactivate = “inactive/hiatus,” not deletion.
- [ ] Inactive quests must remain visible and allow reactivation.

6. Data Independence
- [ ] Save logic must respect data type and destination.
      - Zone data → `zones.json` or localStorage key
      - Lore → `loreBlocks.json`
      - Achievements → `achievements.json`
      - Journal → `questJournal.json`
- [ ] Avoid cross-panel overwrite of content.

## Notes:
- All existing content must eventually be deleted and re-added manually via UI once CRUD logic is verified.
- “Zone Maps” section is foundational — each zone acts as a large quest project.
- “All Zones” visual logic already outlines task status; now ensure selection injects quests properly into journal.
